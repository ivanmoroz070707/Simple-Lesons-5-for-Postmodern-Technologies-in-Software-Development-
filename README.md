# Simple-Lesons-5-for-Postmodern-Technologies-in-Software-Development-
## House API (Laboratory Work #3)
Цей проєкт є RESTful API для управління базою даних нерухомості, написаним на Go з використанням MySQL.  

Це спільна робота, де основна увага приділялася архітектурі, автоматизації міграцій та надійності коду через юніт-тестування.
### Основні функції (Частина Студента 2)
У межах цієї гілки було реалізовано наступний функціонал:
* Автоматичні міграції БД: Додано підтримку golang-migrate. Тепер при старті сервера таблиці створюються автоматично, що позбавляє необхідності вручну запускати SQL-скрипти.
* Часткове оновлення (PATCH): Реалізовано метод PATCH /houses/{id}, який дозволяє оновлювати лише окремі поля (наприклад, тільки ціну), не зачіпаючи інші дані.
* Видалення (DELETE): Додано можливість видалення записів за ID.
* Unit-тестування: Написано тести для хендлерів у файлі handlers_test.go з використанням Mock-об'єктів (бібліотека testify). Це дозволяє тестувати логіку API без реального підключення до бази даних.
* Документація запитів: Створено колекцію для Bruno/Postman, що знаходиться в папці api-docs/.

### Технологічний стек
* Мова: Go (Golang)
* Роутер: Chi Router
* База даних: MySQL
* Міграції: golang-migrate
* Тестування:Testify (Mock)
* API Testing: Bruno / curl

### Структура проєкту
* /api-docs — Колекція запитів у форматі JSON для швидкого тестування.
* /migrations — SQL-файли для автоматичного розгортання схеми БД.
* /models — Опис структур даних (House).
* sql_repository.go — Реалізація логіки роботи з MySQL.
* handlers_test.go — Юніт-тести для перевірки API.

## Як запустити 
1. Налаштування бази даних
   
Переконайтеся, що у вас запущений MySQL. Дані для підключення зчитуються з енвайронменту (налаштовується в config.go). Ви можете використовувати init.sql для початкової підготовки.

3. Запуск сервера
  ```Bash 
   go run .
```
Сервер автоматично запустить міграції та виведе повідомлення:

```2026/05/09 10:16:09 Міграції успішно застосовано!```

``` 2026/05/09 10:16:09 Server is running on port 8080... ```

3. Запуск тестів

Щоб переконатися, що все працює коректно:

``` Bashgo 
test -v .
```
   
## API Ендпоінти
| Метод | Ендпоінт | Опис | 
| :--- | :--- | :--- |
| **GET** | `/houses` | Отримати список усіх будинків | 
| **GET** | `/houses/{id}` | Отримати деталі конкретного будинку | 
| **POST** | `/houses` | Створити новий запис про будинок | 
| **PATCH** | `/houses/{id}` | Частково змінити дані (напр. ціну) |
| **DELETE** | `/houses/{id}` | Видалити будинок з бази даних |

### Тестування в Bruno
1. Встановіть Bruno. Імпортуйте файл api-docs/collection.json.
2. Використовуйте готові запити для перевірки всіх методів.
