# Simple-Lesons-5-for-Postmodern-Technologies-in-Software-Development-

## House API (Laboratory Work #3)
Цей проєкт є RESTful API для управління базою даних нерухомості, написаним на Go з використанням MySQL. 

Це спільна робота, де основна увага приділялася архітектурі, автоматизації міграцій та надійності коду через юніт-тестування.
## 👨‍💻 Зона відповідальності: Студент 1 (Основи, Роутинг та Тестування)

У цьому проєкті було реалізовано базову архітектуру REST API для управління даними про будинки. Основний фокус був на створенні надійного шару обробки HTTP-запитів та їх повному покритті юніт-тестами.

**Основні виконані завдання:**

* **Налаштування архітектури та роутингу:** 
  * Налаштовано базовий сервер та маршрутизацію з використанням стандартного `http.ServeMux` (нові можливості роутингу з Go 1.22).
  * Розроблено структуру моделей (`models.House`) та визначено інтерфейс репозиторію (`HouseRepository`) для подальшої інтеграції з базою даних.

* **Реалізація HTTP-хендлерів (CRUD):**
  * Написано обробники для всього життєвого циклу ресурсу:
    * `POST /houses` — створення нового запису з попередньою валідацією даних.
    * `GET /houses` — отримання списку всіх будинків.
    * `GET /houses/{id}` — отримання конкретного будинку за його ID.
    * `PUT /houses/{id}` — повне оновлення даних будинку.
    * `PATCH /houses/{id}` — часткове оновлення полів.
    * `DELETE /houses/{id}` — видалення запису.

* **Валідація даних:**
  * Додано перевірки вхідних JSON-даних (наприклад, заборона створення будинку без адреси або з нульовою кількістю кімнат) із поверненням відповідних HTTP-статусів (400 Bad Request).
### Основні функції (Частина Студента 2)
У межах цієї гілки було реалізовано наступний функціонал:
* Автоматичні міграції БД: Додано підтримку golang-migrate. Тепер при старті сервера таблиці створюються автоматично, що позбавляє необхідності вручну запускати SQL-скрипти.
* Часткове оновлення (PATCH): Реалізовано метод PATCH /houses/{id}, який дозволяє оновлювати лише окремі поля (наприклад, тільки ціну), не зачіпаючи інші дані.
* Видалення (DELETE): Додано можливість видалення записів за ID.
* Unit-тестування: Написано тести для хендлерів у файлі handlers_test.go з використанням Mock-об'єктів (бібліотека testify). Це дозволяє тестувати логіку API без реального підключення до бази даних.
* Документація запитів: Створено колекцію для Bruno/Postman, що знаходиться в папці api-docs/.
